SCM additional configurations, used to define extra behaviour on multibranch jobs

Fields:
  • organization: String

    Git organization

  • repository: String

    Git repository

  • branches: String

    Wildcard space separated git branch names to build and scan for Pull Requests that target them

  • markerFile: String

    Marker file that makes a branch eligible to be built

  • scanInterval: int

    Branch periodic scan interval in minutes

  • prReportStatus: boolean

    Define if the Pull Request build status should be reported to Github

  • prScan: boolean

    Define if Pull Requests are to be scanned

  • prMerge: boolean

    Define if the Pull Request should be merged with the target branch before built
